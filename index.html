Purpose

A safer bridge between experiments and operation.

Trading tools often split research notebooks, scripts, dashboards, and live controls into separate worlds. QuantaDesk treats them as one evidence pipeline: collect data, test ideas, review risk, observe paper behavior, and only then consider operational use.

The public project is intentionally centered on infrastructure: local data handling, backtest plumbing, operator workflows, reporting, safety checks, and extension points. Private strategies, parameter sets, private market lists, live credentials, and proprietary research artifacts stay outside the open-source boundary.

Usage

Run it as a private local workstation.

The open-source release will be designed for local research first. Use demo strategies and sample data while keeping exchange keys and private strategy modules in your own environment. 

git clone https://github.com/your-org/quantadesk-core.git
cd quantadesk-core

python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev,webui]"

python -m quantadesk.main webui --host 127.0.0.1 --port 8090

Open Boundary

What the public project includes.

  • Local WebUI shell and operator workflow.
  • Backtest engine, run-card format, reports, and safety gates.
  • Risk controls, kill-switch patterns, and audit logging.
  • Demo strategies that explain the interface without real edge.
  • Documentation for writing private strategy plug-ins.

What stays private.

  • Proprietary strategy modules and parameter sets.
  • Research notes, rankings, private market lists, and operational candidates.
  • Market archives, journals, exchange credentials, and operator state.
  • Any evidence artifact that reveals selection logic or live intent.
